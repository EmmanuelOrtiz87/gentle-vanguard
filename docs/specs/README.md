# Specs Directory

This directory is the active workspace for Spec-Driven Development (SDD).

## What Goes Here

1. Active feature specs.
2. Hotfix mini-specs for exception workflows.
3. Specs in progress (`draft`, `reviewed`, `implementing`, `validated`).

## Naming Convention

1. Feature spec: `{feature-slug}.md`
2. Hotfix mini-spec: `hotfix-{slug}-{YYYYMMDD}.md`

## Required Header Fields

1. Status
2. Author
3. Date
4. Linked PR/Branch

## Template

Use `SPEC-TEMPLATE.md` as the baseline for all new specs.

## Status Lifecycle

1. `draft`
2. `reviewed`
3. `implementing`
4. `validated`
5. `done`
