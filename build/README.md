# Gentle-Vanguard Build Scripts

Scripts and tools for encrypting, compiling, and distributing Gentle-Vanguard.

## Quick Build

### Single command — build Gentle-Vanguard.exe

```powershell
# Full build (encrypt + compile installer)
pwsh -NoProfile -ExecutionPolicy Bypass -File "build\create-installer.ps1"

# Skip encryption (if build/protected/ already exists)
pwsh -NoProfile -ExecutionPolicy Bypass -File "build\create-installer.ps1" -SkipEncrypt

# Output: dist\Gentle-Vanguard.exe (~2.4 MB, NSIS installer, AES-256 encrypted)
```

This is the **only** distribution format. `Gentle-Vanguard.exe` is a professional NSIS installer wizard that:

1. Installs to `C:\Program Files\Gentle-Vanguard\`
2. Deploys encrypted scripts (`protected/`) and skill stubs (`public/`)
3. Installs compiled launcher (`Gentle-Vanguard-Launcher.exe`)
4. Creates Desktop and Start Menu shortcuts
5. User provides `master.key` on first launch to decrypt and run

## Prerequisites

| Tool | Purpose | Install Location | Download |
|------|---------|------------------|----------|
| PowerShell 7+ | Script execution | System PATH | Microsoft |
| NSIS 3+ | Compile .exe installer | `C:\Program Files (x86)\NSIS\Bin\makensis.exe` | https://nsis.sourceforge.io/ |
| PS2EXE | Compile PS1 to standalone .exe | PowerShell module | `Install-Module ps2exe -Scope CurrentUser -Force -AllowClobber` |
| Cairo DLL (optional) | SVG to PNG conversion | `tools\cairo.dll` | — |

> **NSIS location**: `C:\Program Files (x86)\NSIS\Bin\makensis.exe` (v3.12). The build script auto-detects it. If NSIS is not found, the build will fail with a clear error.

## Build Pipeline

```
protect-gentle-vanguard.ps1             create-installer.ps1 ← CANONICAL
         |                                    |
   AES-256 encrypt                    Generate .nsi dynamically
   + integrity manifest              + compile launcher (ps2exe)
         |                            + NSIS compile (makensis)
         |                                    |
   build/protected/                    dist/Gentle-Vanguard.exe
   build/public/                      (NSIS installer, AES-256,
   build/integrity-manifest.json       all-in-one, professional wizard)
```

## protect-gentle-vanguard.ps1

Encrypts scripts and configs with AES-256 using the master key.

```powershell
# Full encryption run
pwsh -File build/protect-gentle-vanguard.ps1

# Dry run (no files written)
pwsh -File build/protect-gentle-vanguard.ps1 -DryRun

# Use custom key path
pwsh -File build/protect-gentle-vanguard.ps1 -KeyPath C:\path\to\master.key
```

### What gets encrypted

- **Scripts**: `scripts/adaptive/`, `scripts/common/`, `scripts/diagnostics/`, `scripts/gentle-vanguard/`, `scripts/git-hooks/`, `scripts/hooks/`, `scripts/monitoring/`, `scripts/security/`, `scripts/utilities/`, `scripts/validation/`
- **Configs**: `auto-delegation.json`, `behavior-prompts.json`, `metrics-config.json`, `session-autostart.config.json`, `skill-dependencies.json`, `subagent-mapping.json`
- **Excluded**: Test files (`*.tests.ps1`, `*.test.ps1`), security configs, owner auth

### Output

- `build/protected/` -- AES-256 encrypted files (`.enc` extension)
- `build/public/` -- Skill stubs (SKILL.md only, no code)
- `build/integrity-manifest.json` -- SHA-256 hashes for verification

## create-installer.ps1 (Canonical)

Builds `dist\Gentle-Vanguard.exe` — the single NSIS installer (AES-256 encrypted, all-in-one).

```powershell
# Full build (encrypt + compile)
pwsh -File build/create-installer.ps1

# Skip encryption (already encrypted)
pwsh -File build/create-installer.ps1 -SkipEncrypt

# Dry run
pwsh -File build/create-installer.ps1 -DryRun
```

### Build Options

| Flag | Effect |
|------|--------|
| `-SkipEncrypt` | Skip encryption step (use if `build/protected/` already exists) |
| `-DryRun` | Show what would be done without writing files |

### Installer Behavior

1. Installs to `C:\Program Files\Gentle-Vanguard\`
2. Copies encrypted scripts to `protected/`
3. Copies skill stubs to `public/`
4. Creates `keys/HOW_TO_GET_KEY.txt` with instructions
5. Creates Desktop and Start Menu shortcuts
6. User must place `master.key` in `keys/` to decrypt and run

### What's inside Gentle-Vanguard.exe

| Component | Source | Destination |
|-----------|--------|-------------|
| Gentle-Vanguard-Launcher.exe | `build/compiled/` | `C:\Program Files\Gentle-Vanguard\` |
| Encrypted scripts | `build/protected/` | `C:\Program Files\Gentle-Vanguard\protected\` |
| Skill stubs | `build/public/` | `C:\Program Files\Gentle-Vanguard\public\` |
| Master key instructions | generated | `C:\Program Files\Gentle-Vanguard\keys\HOW_TO_GET_KEY.txt` |

## Important Notes

1. **`dist\Gentle-Vanguard.exe`** is the **only** distribution format — NSIS installer, AES-256 encrypted, all-in-one
2. `dist\Gentle-Vanguard-Setup.exe` and `build\compiled\Gentle-Vanguard-Launcher.exe` are **deprecated** (removed from sync, use Gentle-Vanguard.exe)
3. After modifying `Gentle-Vanguard-Launcher.ps1`, rebuild with: `pwsh -File build/create-installer.ps1`
4. **`master.key`** is NEVER distributed — users must obtain it from the private repo
5. **`build/`** is in `.gitignore` except for `Gentle-Vanguard-Launcher.ps1` which is force-tracked
6. **`gentle-vanguard-installer-auto.nsi`** is auto-generated — do not edit manually

## End-to-End Build & Deploy

Complete flow from code change to public release:

```powershell
# 1. Make code changes in private repo (develop branch)
# 2. Commit and push
git add . && git commit -m "fix: description" && git push origin develop

# 3. Create PR and merge to main
gh pr create --base main --head develop --title "fix: description"
gh pr merge <PR_NUMBER> --merge --delete-branch

# 4. Switch to main and pull
git checkout main && git pull origin main

# 5. Build the .exe (encrypt + compile NSIS installer)
pwsh -NoProfile -ExecutionPolicy Bypass -File "build\create-installer.ps1"
# Output: dist\Gentle-Vanguard.exe (~2.75 MB)

# 6. Sync to public repo (copies .exe, docs, CI scripts, cleans up)
pwsh -NoProfile -ExecutionPolicy Bypass -File "scripts\utilities\DEPLOYMENT\sync-to-public.ps1"
# This auto-commits and pushes to gentle-vanguard-public
```

### Quick Reference

| Step | Command | Output |
|------|---------|--------|
| Build .exe | `pwsh -File build/create-installer.ps1` | `dist/Gentle-Vanguard.exe` |
| Build (skip encrypt) | `pwsh -File build/create-installer.ps1 -SkipEncrypt` | `dist/Gentle-Vanguard.exe` |
| Build (dry run) | `pwsh -File build/create-installer.ps1 -DryRun` | Preview only |
| Sync to public | `pwsh -File scripts/utilities/DEPLOYMENT/sync-to-public.ps1` | Pushes to gentle-vanguard-public |
| Sync (no push) | `pwsh -File scripts/utilities/DEPLOYMENT/sync-to-public.ps1 -skipPush` | Local copy only |

## Encryption Format

Protected scripts use AES-256 encryption:

- **Key**: 32-byte random key stored in `keys/master.key`
- **Format**: `Base64(IV[16 bytes] + encrypted_data)`
- **Decryption**: Use raw AES (NOT `ConvertTo-SecureString` which causes Windows credential popups)
- **Launcher**: `Gentle-Vanguard-Launcher.ps1` decrypts at runtime and caches to `$env:LOCALAPPDATA\Gentle-Vanguard\scripts\`
