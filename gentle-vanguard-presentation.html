CALIDAD

Judgment Day

Protocolo de revisión dual adversarial. Dos jueces AI ciegos evalúan en paralelo, sintetizan, ofrecen fix cycle y re-evalúan.

Juez A Juez B → Síntesis → Fix Cycle → Re-Juicio → APPROVED → ESCALATED

⚠ Ejecución Automática

  • Pre-push hook: juzga cambios antes de push
  • Pre-merge-commit: juzga antes de merge PR
  • GitHub Actions: semanal (lun 10:00 ART)
  • Evento session.ended también lo dispara

📜 Reglas Duras

  • Max 2 iteraciones de fix
  • Nunca declarar APPROVED si hay CRITICALs
  • Preservar posiciones minoritarias
  • Detectar sycophancy (cambio sin evidencia)

🎯 Gobernanza

  • Bloquea merge si judgment falla
  • Labels auto: approved/failed/pending
  • Coordina con CI pipeline
  • Reporta al event bus
