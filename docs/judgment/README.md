# Judgment

This directory stores Judgment Day outputs and reporting artifacts.

## Purpose

1. Keep adversarial review evidence and verdict history.
2. Record dashboard-friendly metrics and final decisións.
3. Support release-readiness and risk reviews.

## Typical Files

- `dashboard.md` - rendered judgment dashboard
- `dashboard.csv` - tabular history for trend analysis
- `draft-report.md` - intermediate report output
- `final-verdict.md` - final release/readiness decisión
