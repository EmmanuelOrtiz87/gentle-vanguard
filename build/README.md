# Foundation Build Scripts

Scripts and tools for encrypting, compiling, and distributing Foundation.

## Quick Build Options

### Option A: Single-file executable (recommended)

No installation needed. Download `Foundation.exe` and run it. AES-256 encrypted — master.key required on first run, cached afterwards.

```powershell
pwsh -NoProfile -ExecutionPolicy Bypass -File "build\sfx-build.ps1"
# Output: dist\Foundation.exe (~2.7 MB, AES-256 encrypted, self-contained)
```

### Option B: Encrypted installer distribution

For distributing Foundation with AES-256 encrypted scripts and `master.key` authentication:

```powershell
# 1. Encrypt all protected scripts
pwsh -NoProfile -ExecutionPolicy Bypass -File "build\protect-foundation.ps1"

# 2. Generate NSIS script and compile installer
pwsh -NoProfile -ExecutionPolicy Bypass -File "build\create-installer.ps1"

# 3. Installer output: dist\Foundation-Setup.exe
```

## Prerequisites

| Tool | Purpose | Download |
|------|---------|----------|
| PowerShell 7+ | Script execution | Microsoft |
| NSIS 3+ | Compile .exe installer | https://nsis.sourceforge.io/ |
| PS2EXE (optional) | Compile PS1 to standalone .exe | `Install-Module ps2exe` |
| Cairo DLL (optional) | SVG to PNG conversion | `tools\cairo.dll` |

## Build Pipeline

```
sfx-build.ps1                    protect-foundation.ps1          create-installer.ps1
     |                                  |                              |
  ZIP 264 files                  AES-256 encrypt              Generate .nsi
  Base64 embed                   + compile launcher            + compile launcher
  ps2exe compile                 + build .bat wrapper          + NSIS compile
     |                                  |                              |
  dist/Foundation.exe           build/protected/              dist/Foundation-Setup.exe
  (self-contained,              build/public/                  (encrypted installer,
   no install)                  build/integrity-manifest.json   requires master.key)
```

| File | Purpose |
|------|---------|

## protect-foundation.ps1

Encrypts scripts and configs with AES-256 using the master key.

```powershell
# Full encryption run
pwsh -File build/protect-foundation.ps1

# Dry run (no files written)
pwsh -File build/protect-foundation.ps1 -DryRun

# Use custom key path
pwsh -File build/protect-foundation.ps1 -KeyPath C:\path\to\master.key
```

### What gets encrypted

- **Scripts**: `scripts/adaptive/`, `scripts/common/`, `scripts/diagnostics/`, `scripts/foundation/`, `scripts/git-hooks/`, `scripts/hooks/`, `scripts/monitoring/`, `scripts/security/`, `scripts/utilities/`, `scripts/validation/`
- **Configs**: `auto-delegation.json`, `behavior-prompts.json`, `metrics-config.json`, `session-autostart.config.json`, `skill-dependencies.json`, `subagent-mapping.json`
- **Excluded**: Test files (`*.tests.ps1`, `*.test.ps1`), security configs, owner auth

### Output

- `build/protected/` -- AES-256 encrypted files (`.enc` extension)
- `build/public/` -- Skill stubs (SKILL.md only, no code)
- `build/integrity-manifest.json` -- SHA-256 hashes for verification

## create-installer.ps1

Generates the NSIS installer script and compiles the .exe.

```powershell
# Full build (encrypt + compile)
pwsh -File build/create-installer.ps1

# Skip encryption (already encrypted)
pwsh -File build/create-installer.ps1 -SkipEncrypt

# Dry run
pwsh -File build/create-installer.ps1 -DryRun
```

### Build Options

| Flag | Effect |
|------|--------|
| `-SkipEncrypt` | Skip encryption step (use if `build/protected/` already exists) |
| `-DryRun` | Show what would be done without writing files |

### Installer Behavior

1. Installs to `C:\Program Files\Foundation\`
2. Copies encrypted scripts to `protected/`
3. Copies skill stubs to `public/`
4. Creates `keys/HOW_TO_GET_KEY.txt` with instructions
5. Creates Desktop and Start Menu shortcuts
6. User must place `master.key` in `keys/` to decrypt and run

## sfx-build.ps1

Builds the single-file `Foundation.exe` (AES-256 encrypted, self-contained, no installation needed).

```powershell
# Build (requires build/protected/ from protect-foundation.ps1)
pwsh -File build/sfx-build.ps1

# Dry run
pwsh -File build/sfx-build.ps1 -DryRun
```

### How it works

1. Collects encrypted `.enc` files from `build/protected/` (263 files, ~2.6 MB)
2. Compresses into ZIP (~2.0 MB)
3. Base64-encodes the ZIP (~2.6 MB)
4. Embeds the Base64 data into `Foundation-Launcher.ps1` (replaces placeholder `__EMBEDDED_SCRIPTS__`)
5. Compiles with ps2exe into `dist\Foundation.exe` (~2.7 MB)

### On first run

1. Extracts encrypted `.enc` files from embedded archive to `%TEMP%\Foundation\embedded\`
2. Searches for `master.key`: local cache → next to exe → user prompt (one time)
3. Decrypts `.enc` files with AES-256 and caches scripts to `%LOCALAPPDATA%\Foundation\scripts\`
4. Key is cached to `%LOCALAPPDATA%\Foundation\data\master.key` for subsequent runs
5. Executes `wf.ps1` from cache

### Security

- **Scripts are NEVER stored in plaintext inside the exe**. Only AES-256 encrypted `.enc` files are embedded.
- **master.key is required** to decrypt on first run. After that, the key is cached locally so user doesn't need to re-enter.
- **Same encryption** as the installed distribution (`build/protected/`).
- Anyone with the exe CANNOT extract the scripts without the master.key.

### Output

- `dist\Foundation.exe` — single file, AES-256 encrypted, master.key required on first run

## Important Notes

1. **`dist\Foundation.exe`** is the recommended distribution — single file, no installation
2. **`dist\Foundation-Setup.exe`** is the encrypted installer (requires `master.key`)
3. After modifying `Foundation-Launcher.ps1`, rebuild both exes:
   - `pwsh -File build/sfx-build.ps1` (for Foundation.exe)
   - `pwsh -File build/create-installer.ps1` (for Foundation-Setup.exe)
4. **`master.key`** is NEVER distributed — users must obtain it from the private repo
5. **`build/`** is in `.gitignore` except for `Foundation-Launcher.ps1` which is force-tracked
6. **`foundation-installer-auto.nsi`** is auto-generated — do not edit manually

## Encryption Format

Protected scripts use AES-256 encryption:

- **Key**: 32-byte random key stored in `keys/master.key`
- **Format**: `Base64(IV[16 bytes] + encrypted_data)`
- **Decryption**: Use raw AES (NOT `ConvertTo-SecureString` which causes Windows credential popups)
- **Launcher**: `Foundation-Launcher.ps1` decrypts at runtime and caches to `$env:LOCALAPPDATA\Foundation\scripts\`
